**Дата**: 2026-04-08
**Ветка**: `002-health-data-entry`

## Предварительные требования

| Зависимость | Минимальная версия |
|-------------|-------------------|
| Node.js | 20.x+ |
| Rust | 1.75+ |
| make | GNU Make 3.81+ |

## Установка

```bash
# Из корня проекта
make install
```

## Запуск в режиме разработки

```bash
make dev
```

## Использование

### Внесение замера

1. Откройте приложение → перейдите в раздел «Здоровье» (меню навигации)
2. Нажмите **«Новый замер»** — появится столбец слева
3. Укажите дату (по умолчанию — сегодня)
4. Заполните показатели — клик по ячейке открывает ввод
5. Показатели с автозаполнением (рост) копируются из предыдущего замера
6. При вводе отображается предыдущее значение и отклонение

### Редактирование

- Клик по любой ячейке — режим редактирования
- Enter или клик вне ячейки — сохранение
- Backspace/Delete в пустой ячейке — очистка значения
- Контекстное меню заголовка столбца → «Удалить замер»

### Графики

1. Переключитесь в режим **«Графики»** (табы вверху)
2. Отметьте нужные показатели чекбоксами
3. Перемещайте ползунки на временной шкале для фильтрации периода
4. По умолчанию — весь доступный период

### Расширение показателей

Добавьте новый показатель в `metric-config.json` (в директории данных приложения):
1. Откройте файл `metric-config.json`
2. Добавьте объект MetricDefinition в массив `metrics`
3. Укажите `key`, `label`, `unit`, `type`, `range`, `autofill`, `order`, `category`
4. Перезапустите приложение

## Структура данных

```
{APP_DATA_DIR}/
├── health.json           # Замеры здоровья
├── metric-config.json    # Конфигурация показателей
└── logs/
    └── app-*.log         # Логи (при debug: true)
```

## Конфигурация логирования

Установите `"debug": true` в `config.json` для детального логирования IPC-операций.
