## Сводка

Новый модуль «Препараты» для десктопного приложения my-health. Позволяет пользователю вводить и просматривать информацию о принимаемых лекарственных препаратах и БАДах. Ключевые возможности: реестр препаратов с фильтрацией по активности и категориям, детальная карточка с индикатором статуса «принимается сейчас», модальная форма с автодополнением текстовых полей из ранее введённых значений. Интерфейс аналогичен модулям «Приёмы» и «Анализы».

## Технический контекст

**Язык/Версия**: TypeScript 5.x (Frontend), Rust (Tauri backend)
**Основные зависимости**: React, shadcn/ui, Tailwind CSS, zod (валидация), @tauri-apps/api, существующие модули doctor-visits и lab-tests как референсы
**Хранилище**: JSON-файлы (`medications.json`) в файловой системе
**Тестирование**: Vitest (unit-тесты frontend), cargo test (unit-тесты Rust)
**Целевая платформа**: Windows, Linux, macOS (Tauri desktop)
**Тип проекта**: Десктопное приложение (Tauri + React)
**Цели производительности**: Загрузка реестра <500мс при 500 записях, рендеринг карточки <200мс при 100 записях
**Ограничения**: Все данные локально; текстовые поля с лимитами (наименование ≤ 200, категория ≤ 100, действующее вещество ≤ 200, дозировка ≤ 100, кратность ≤ 100, допинформация ≤ 500 символов)
**Масштаб/Объем**: 3 экранные формы (реестр, карточка, модалка), ~10 новых компонентов, сервис, Tauri-команды, утилиты, unit-тесты

### Исходный код (корень репозитория)

```text
src/
├── components/
│   └── medications/
│       ├── MedicationView.tsx            # Главный экран (реестр + модалка)
│       ├── MedicationRegistry.tsx        # Реестр с фильтрацией
│       ├── MedicationTile.tsx            # Плитка препарата в реестре
│       ├── MedicationDetailPage.tsx      # Детальная карточка
│       ├── MedicationCard.tsx            # Полная карточка препарата
│       ├── MedicationModal.tsx           # Форма создания/редактирования
│       ├── MedicationFilters.tsx         # Фильтры (переключатель, категория, наименование)
│       └── MedicationAutocomplete.tsx    # Автодополнение для текстовых полей
├── services/
│   └── medication-service.ts             # IPC-сервис (CRUD)
│   └── __tests__/
│       └── medication-service.test.ts
├── lib/
│   ├── medication-utils.ts               # Фильтрация, сортировка, навигация, автодополнение
│   └── __tests__/
│       └── medication-utils.test.ts
├── types/
│   └── index.ts                          # Medication, MedicationCategory
├── contracts/
│   └── medication-commands.ts            # IPC-команды (типы запросов/ответов)

src-tauri/
├── src/
│   ├── storage/
│   │   └── medication_store.rs           # Хранилище JSON
│   └── commands/
│       └── medications.rs                # Tauri-команды (CRUD)
```

**Решение по структуре**: Один проект (Tauri + React). Модуль `medications` следует паттерну `doctor-visits` и `lab-tests` — те же слои: компоненты → сервис → утилиты → типы → Rust-хранилище. Нет внешних сканов/файлов — только JSON.
