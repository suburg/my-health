# Модель данных: Модуль «Препараты»

**Дата**: 2026-04-10
**Ветка**: `005-medications`

## Сущности

### Medication (Препарат)

Представляет одну запись о принимаемом препарате или БАДе.

**Файл хранения**: `%APPDATA%/com.myhealth.app/medications.json`

| Поле | Тип | Обязательный | Макс. длина | Описание |
|------|-----|:-----------:|:-----------:|----------|
| id | string (UUID v4) | Да | — | Уникальный идентификатор |
| name | string | Да | 200 | Наименование препарата |
| category | string | Да | 100 | Категория (произвольный текст) |
| activeIngredient | string \| null | Нет | 200 | Действующее вещество (МНН) |
| dosage | string | Да | 100 | Дозировка (например, «500 мг») |
| frequency | string | Да | 100 | Кратность приёма (например, «2 раза в день») |
| startDate | string (ДД.ММ.ГГГГ) | Да | — | Дата начала приёма |
| endDate | string (ДД.ММ.ГГГГ) \| null | Нет | — | Дата окончания (null = принимается сейчас) |
| notes | string \| null | Нет | 500 | Дополнительная информация |
| createdAt | string (ISO 8601) | Да | — | Дата создания записи |
| updatedAt | string (ISO 8601) | Да | — | Дата последнего изменения |

```json
{

      "createdAt": "2026-04-01T10:00:00.000Z",
      "updatedAt": "2026-04-01T10:00:00.000Z"
    }
  ]
}
```

## Правила валидации

### Medication
- `name` — непустая строка, макс. 200 символов
- `category` — непустая строка, макс. 100 символов
- `dosage` — непустая строка, макс. 100 символов
- `frequency` — непустая строка, макс. 100 символов
- `startDate` — корректная дата в формате ДД.ММ.ГГГГ
- `endDate` — если указана: корректная дата в формате ДД.ММ.ГГГГ, не раньше `startDate`
- `activeIngredient` — если указан: макс. 200 символов
- `notes` — если указан: макс. 500 символов

## Связи между сущностями

```
Medication ──1:1──> Medication (нет внешних связей)
```

Модуль не имеет внешних связей. Навигация prev/next вычисляется динамически по категории и дате.

## Статус «Принимается сейчас»

Вычисляемое поле, не хранится в JSON:

```
isActive(medication) = medication.endDate === null || medication.endDate >= today
```

## Миграции / Версионирование

- Поле `schemaVersion` в корне JSON-файла
- Текущая версия: `1`
- При изменении структуры — инкремент версии + обработчик миграции
