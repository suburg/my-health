# План реализации: Ввод и визуализация данных о здоровье

**Ветка**: `002-health-data-entry` | **Дата**: 2026-04-08 | **Спецификация**: [spec.md](./spec.md)
**Входные данные**: Спецификация функции из `/specs/002-health-data-entry/spec.md`

## Сводка

Создание модуля журнала здоровья в десктопном Tauri-приложении: таблица-реестр (строки — показатели, столбцы — замеры по датам) с inline-редактированием, автозаполнением показателей из предыдущего замера, расчётом отклонений, переключением в режим графиков с фильтрацией по периоду (ползунки на временной шкале). Данные хранятся в JSON-файлах. Набор показателей расширяется через конфигурацию.

## Технический контекст

**Язык/Версия**: TypeScript 5.x
**Основные зависимости**: Tauri 2.x, React 19, Tailwind CSS, shadcn/ui, zod (валидация), библиотека графиков Recharts
**Хранилище**: JSON-файлы в файловой системе (локально, атомарная запись)
**Тестирование**: Vitest (unit-тесты бизнес-логики), Playwright или Tauri test (E2E)
**Целевая платформа**: Windows, Linux, macOS (десктоп)
**Тип проекта**: десктопное приложение (Tauri + React SPA) — модуль к существующему приложению
**Цели производительности**: отклик inline-редактирования < 50 мс, перерисовка графиков < 200 мс, таблица до 100+ замеров без лагов
**Ограничения**: полностью оффлайн; таблица с горизонтальной прокруткой при >10 замерах; фиксация первого столбца
**Масштаб/Объем**: ~11 показателей, неограниченное количество замеров, 2 JSON-файла (замеры + конфигурация показателей)

│   │   │                    #               get_entries, get_entry_by_date
│   │   └── mod.rs           # Экспорт команд

│   │   ├── HealthCharts.tsx     # Графики с ползунками периода
│   │   ├── MetricSelector.tsx   # Чекбоксы выбора показателей
│   │   ├── PeriodFilter.tsx     # Ползунки начала/конца периода

│   └── validations.ts         # Zod-схемы для health-данных

**Решение по структуре**: Существующее Tauri-приложение расширяется новым модулем. Бэкенд — новые IPC-команды в `commands/health.rs`, хранилище — `health_store.rs` поверх существующего `json_store.rs`. Фронтенд — группа компонентов в `components/health/`, сервис `health-service.ts`, конфигурация показателей `metric-config.ts`. Данные замеров — отдельный JSON-файл `health.json` в appDataDir.
