# Спецификация функции: Модуль «Препараты»

**Ветка функции**: `005-medications`
**Создана**: 2026-04-10
**Статус**: Черновик
**Входные данные**: Описание пользователя: Препараты – ввод информации о принимаемых лекарственных препаратах и БАДах. Реестр аналогичный реестру приёмов и реестру анализов, заимствуй из них компоновку и интерфейсные решения. Состав информации: период приёма, категория, наименование препарата, действующее вещество, дозировка, кратность приёма, дополнительная информация. Реестр, форма добавления препарата, изменения, карточка. Фильтры в реестре – переключатель «принимаемые сейчас», категория.

## Сценарии пользователя и тестирование *(обязательно)*

### Пользовательская история 1 — Просмотр реестра препаратов

Пользователь открывает модуль «Препараты» и видит список всех записей о препаратах, отсортированный по дате начала приёма (новые сверху). Каждая запись отображается в виде карточки-плитки с ключевой информацией: наименование препарата, категория, период приёма, индикатор «принимается сейчас». Пользователь может фильтровать записи переключателем «Принимаемые сейчас», фильтром по категории и фильтром по наименованию препарата.

**Почему этот приоритет**: Базовый сценарий — пользователь должен видеть все свои препараты в одном месте. Без реестра остальные функции бесполезны. Это MVP модуля.

**Независимое тестирование**: Можно полностью протестировать, добавив несколько записей препаратов через seed-данные и проверив их отображение, сортировку и фильтрацию в реестре.

**Сценарии приёма**:

1. **Дано** в системе есть 5 записей препаратов с разными датами начала, **Когда** пользователь открывает модуль «Препараты», **Тогда** отображаются все 5 записей, отсортированные по убыванию даты начала
2. **Дано** пользователь включил переключатель «Принимаемые сейчас», **Когда** реестр обновляется, **Тогда** отображаются только препараты без даты окончания или с датой окончания в будущем
3. **Дано** в реестре есть препараты разных наименований, **Когда** пользователь вводит начало названия в фильтре, **Тогда** показываются только совпадающие препараты

---

### Пользовательская история 2 — Ручной ввод препарата

Пользователь создаёт новую запись препарата: указывает наименование, категорию (лекарство, БАД, витамин, трава, прочее), действующее вещество, дозировку, кратность приёма, дату начала и опционально дату окончания приёма, дополнительную информацию. Система определяет статус «принимается сейчас» на основе дат.

**Почему этот приоритет**: Основной способ наполнения модуля данными. Без ручного ввода пользователь не может вести учёт препаратов.

**Независимое тестирование**: Можно полностью протестировать, создав запись препарата вручную с заполнением всех полей, проверив валидацию, сохранение и корректное отображение в реестре и карточке.

**Сценарии приёма**:

1. **Дано** пользователь открыл форму создания препарата, **Когда** заполняет обязательные поля (наименование, категория, дата начала, дозировка, кратность) и нажимает «Сохранить», **Тогда** запись появляется в реестре
2. **Дано** пользователь не указал дату окончания приёма, **Когда** запись сохраняется, **Тогда** препарат считается «принимаемым сейчас»
3. **Дано** пользователь указал дату окончания в прошлом, **Когда** запись отображается в реестре, **Тогда** индикатор «принимается сейчас» отсутствует

---

### Пользовательская история 3 — Просмотр детальной карточки препарата

Пользователь кликает на запись в реестре и переходит на детальную страницу препарата. Карточка содержит: наименование, категорию, действующее вещество, дозировку, кратность приёма, период приёма с визуальным индикатором статуса, дополнительную информацию. Доступна навигация к предыдущему/следующему препарату той же категории, а также редактирование и удаление.

**Почему этот приоритет**: Завершающий пользовательский опыт — детальный просмотр и управление. Зависит от историй 1 и 2, но ценен сам по себе.

**Независимое тестирование**: Можно протестировать, открыв детальную карточку существующего препарата, проверив отображение всех полей, навигацию и функции редактирования/удаления.

**Сценарии приёма**:

1. **Дано** пользователь кликнул на запись в реестре, **Когда** открывается детальная страница, **Тогда** отображаются все данные препарата
2. **Дано** препарат принимается сейчас (нет даты окончания или она в будущем), **Когда** отображается карточка, **Тогда** показывается визуальный индикатор «Принимается сейчас»
3. **Дано** пользователь нажимает «Редактировать», **Когда** открывается форма, **Тогда** все данные предзаполнены и доступны для изменения

---

### Граничные случаи

- **Пустой реестр**: Пользователь впервые открыл модуль — отображается пустое состояние с подсказкой «Добавьте первый препарат»
- **Дата окончания раньше даты начала**: Система не позволяет сохранить такую запись — отображается ошибка валидации
- **Препарат без дозировки**: Дозировка обязательна для сохранения
- **Удаление препарата**: При удалении пользователь должен подтвердить действие (модальное окно подтверждения)
- **Дублирование названий**: Пользователь может добавить несколько записей одного и того же препарата с разными периодами или дозировками — система не блокирует это

### Форма 1: Реестр препаратов (MedicationRegistry)

**Статус**: НОВАЯ
**Связанная история**: Пользовательская история 1

**Элементы управления**:

| Элемент | Тип | Обязательный | Описание |
|---------|-----|:-----------:|----------|
| Кнопка «Добавить препарат» | Кнопка | Да | Открывает форму создания препарата |
| Переключатель «Принимаемые сейчас» | Toggle/чекбокс | Нет | Фильтр: показывает только активные препараты |
| Фильтр по наименованию | Текстовое поле с автодополнением | Нет | Ввод или выбор из ранее введённых наименований |
| Фильтр по категории | Текстовое поле с автодополнением | Нет | Ввод или выбор из ранее введённых категорий |
| Карточка-плитка препарата | Карточка (клик) | Да | Отображает: наименование, категорию, период, индикатор «принимается сейчас» |
| Пустое состояние | Placeholder | Да | Сообщение при отсутствии записей |

**Логика работы**:

При открытии формы загружаются все препараты из хранилища, сортируются по убыванию даты начала приёма. Отображаются в виде адаптивной сетки карточек-плиток. Переключатель «Принимаемые сейчас» работает мгновенно. Фильтры по наименованию и категории работают по вводу с автодополнением из ранее введённых значений. Клик по плитке открывает детальную страницу препарата.

**Статус**: НОВАЯ
**Связанная история**: Пользовательская история 3

### Форма 3: Форма создания/редактирования препарата (MedicationModal)

**Статус**: НОВАЯ
**Связанная история**: Пользовательская история 2

**Элементы управления**:

| Элемент | Тип | Обязательный | Описание |
|---------|-----|:-----------:|----------|
| Наименование препарата | Текстовое поле с автодополнением | Да | Ввод или выбор из ранее введённых наименований |
| Категория | Текстовое поле с автодополнением | Да | Ввод или выбор из ранее введённых категорий |
| Действующее вещество | Текстовое поле с автодополнением | Нет | Ввод или выбор из ранее введённых МНН |
| Дозировка | Текстовое поле | Да | Например: «500 мг», «1 таблетка» |
| Кратность приёма | Текстовое поле | Да | Например: «2 раза в день», «утром натощак» |
| Дата начала приёма | Поле даты (ДД.ММ.ГГГГ) | Да | Дата начала приёма препарата |
| Дата окончания приёма | Поле даты (ДД.ММ.ГГГГ) | Нет | Если не указано — препарат принимается сейчас |
| Дополнительная информация | Текстовое поле (многострочное) | Нет | Примечания, особенности приёма |
| Кнопка «Сохранить» | Кнопка | Да | Сохраняет препарат (с валидацией) |
| Кнопка «Отмена» | Кнопка | Да | Закрывает форму без сохранения |

**Логика работы**:

Форма открывается в режиме создания (пустая) или редактирования (предзаполненная). Обязательные поля: наименование, категория, дозировка, кратность приёма, дата начала. Поля «Наименование», «Категория» и «Действующее вещество» поддерживают автодополнение по ранее введённым значениям. Валидация: дата окончания не может быть раньше даты начала. Сохранение через сервис с атомарной записью.

## Требования *(обязательно)*

### Функциональные требования

- **FR-001**: Система ДОЛЖНА предоставлять реестр всех записей о препаратах с сортировкой по дате начала приёма (новые сверху)
- **FR-002**: Система ДОЛЖНА позволять фильтровать препараты переключателем «Принимаемые сейчас» (без даты окончания или дата окончания ≥ сегодня), по наименованию и по категории. Все фильтры комбинируются (логика AND) — результат показывает записи, удовлетворяющие всем активным фильтрам одновременно
- **FR-003**: Система ДОЛЖНА позволять создавать новые записи препаратов через модальную форму
- **FR-004**: Система ДОЛЖНА позволять редактировать и удалять существующие записи препаратов
- **FR-005**: Каждая запись препарата ДОЛЖНА содержать: наименование (≤ 200 символов), категорию (≤ 100 символов, произвольный текст), действующее вещество (≤ 200 символов, опционально), дозировку (≤ 100 символов), кратность приёма (≤ 100 символов), дату начала, дату окончания (опционально), дополнительную информацию (≤ 500 символов, опционально)
- **FR-006**: Система ДОЛЖНА автоматически определять статус «принимается сейчас» на основе дат: если дата окончания не указана или ≥ сегодня — препарат считается активным
- **FR-007**: Система ДОЛЖНА визуально отображать статус «Принимается сейчас» в реестре и карточке (цветовой индикатор)
- **FR-008**: При сохранении записи дата окончания НЕ ДОЛЖНА быть раньше даты начала
- **FR-009**: Система ДОЛЖНА хранить данные препаратов в JSON-файле с полем `schemaVersion` для версионирования структуры
- **FR-010**: При удалении препарата система ДОЛЖНА требовать подтверждения пользователя
- **FR-011**: Система ДОЛЖНА корректно обрабатывать отсутствие, повреждение или несовместимую версию файла данных препаратов
- **FR-012**: Система ДОЛЖНА поддерживать навигацию «предыдущий» / «следующий» препарат в рамках одной категории
- **FR-013**: Система ДОЛЖНА предоставлять автодополнение для полей «Наименование», «Категория» и «Действующее вещество» — предлагать уникальные значения из ранее введённых записей

### Ключевые сущности

- **Medication (Препарат)**: Представляет одну запись о принимаемом препарате или БАДе. Ключевые атрибуты: ID, наименование, категория (произвольный текст), действующее вещество, дозировка, кратность приёма, дата начала, дата окончания (опционально), дополнительная информация, createdAt, updatedAt.

## Допущения

- **А1**: Приложение работает в одно-пользовательском режиме — все препараты принадлежат одному пользователю (соответствует конституции)
- **А2**: Все данные хранятся локально в JSON-файлах, без сетевой синхронизации (соответствует конституции)
- **А3**: Интерфейс модуля следует паттерну модулей «Приёмы» и «Анализы» — реестр с плитками, модальная форма, детальная карточка
- **А4**: Переключатель «Принимаемые сейчас» работает мгновенно, без кнопки «Применить»
- **А5**: Пользователь может добавить несколько записей одного и того же препарата с разными периодами — система не блокирует дублирование наименований
- **А6**: Дата окончания опциональна — если не указана, препарат считается принимаемым в настоящий момент
- **А7**: Визуальная индикация статуса использует зелёный цвет для активных препаратов, серый — для завершённых (в соответствии со светлой темой приложения)
- **А8**: Автодополнение для наименования, категории и действующего вещества работает по аналогии с автодополнением в модуле «Приёмы» (VisitAutocomplete) — предлагает ранее введённые уникальные значения
