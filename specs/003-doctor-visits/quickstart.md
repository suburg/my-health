# Быстрый старт: Ведение информации о приёмах врача

## Запуск приложения

```powershell
# Установка зависимостей (если ещё не установлены)
npm install

# Запуск в режиме разработки
npm run tauri dev
```

## Конфигурация LLM-провайдера (опционально)

Для работы распознавания сканов настройте переменные окружения **перед запуском**:

### Вариант 1: OpenAI (удалённый)

```powershell
$env:LLM_API_URL = "https://api.openai.com/v1"
$env:LLM_API_KEY = "sk-ваш_ключ"
$env:LLM_MODEL = "gpt-4o"
npm run tauri dev
```

### Вариант 2: Ollama (локальный)

Убедитесь, что Ollama запущена и модель `llava` загружена:

```powershell
# Проверьте, что Ollama работает
curl http://localhost:11434

# Загрузите модель llava (если ещё не загружена)
ollama pull llava

# Запустите приложение
$env:LLM_API_URL = "http://localhost:11434/v1"
$env:LLM_API_KEY = "unused"
$env:LLM_MODEL = "llava"
npm run tauri dev
```

### Вариант 3: Без LLM

Если переменные окружения не заданы — распознавание сканов недоступно, но ручной ввод работает полностью.

## Тестирование функции

1. Зарегистрируйтесь/войдите в приложение
2. В боковом меню выберите **«Приёмы»**
3. Нажмите **«Добавить приём»**
4. Заполните обязательные поля (дата, ФИО врача, специальность)
5. Нажмите **«Сохранить»** — запись появится в реестре
6. Кликните на плитку — откроется карточка приёма

### Тестирование LLM-распознавания

1. Откройте форму добавления приёма
2. Загрузите изображение или многостраничный PDF медицинского документа
3. Нажмите **«Распознать скан»**
4. Дождитесь заполнения полей (обрабатываются все страницы документа)
5. Проверьте и отредактируйте распознанные данные
6. Нажмите **«Сохранить»**

## Где находятся данные

- **JSON-файл**: `%APPDATA%\com.myhealth.app\doctor-visits.json`
- **Сканы**: `%APPDATA%\com.myhealth.app\scans\` (имена с бизнес-данными: `2026_04_09_Кардиолог_a1b2c3d4.pdf`)
- **Промпт LLM**: `%APPDATA%\com.myhealth.app\llm-prompt.json` — можно редактировать без пересборки. Изменения вступают в силу при следующем вызове распознавания
