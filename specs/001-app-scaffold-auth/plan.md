# План реализации: Каркас приложения и регистрация пользователя

**Ветка**: `001-app-scaffold-auth` | **Дата**: 2026-04-07 | **Спецификация**: [spec.md](./spec.md)
**Входные данные**: Спецификация функции из `/specs/001-app-scaffold-auth/spec.md`

## Сводка

Создание каркаса десктопного приложения на Tauri + React + TypeScript с регистрацией однопользовательского профиля (фамилия, имя, дата рождения, пол), аутентификацией по пин-коду (4 цифры), основным экраном с меню навигации, просмотром/редактированием профиля, сменой пин-кода и функцией выхода. Makefile для сборки и запуска. Данные хранятся локально в JSON-файлах.

## Технический контекст

**Язык/Версия**: TypeScript 5.x
**Основные зависимости**: Tauri 2.x, React 19, Tailwind CSS, shadcn/ui (Radix UI + class-variance-authority), zod (валидация), argon2 или bcrypt (хэширование пин-кода — через Tauri/Rust)
**Хранилище**: JSON-файлы в файловой системе (локально, атомарная запись)
**Тестирование**: Vitest (unit-тесты бизнес-логики), Playwright или Tauri test (E2E)
**Целевая платформа**: Windows, Linux, macOS (десктоп)
**Тип проекта**: десктопное приложение (Tauri + React SPA)
**Цели производительности**: запуск приложения < 3 секунд, отклик UI < 100 мс
**Ограничения**: полностью оффлайн, без сетевой активности; один пользователь; пин-код — единственный метод входа
**Масштаб/Объем**: 5 экранных форм, 2 JSON-файла (профиль + конфигурация), Makefile
