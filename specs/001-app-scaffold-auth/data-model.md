# Data Model: Каркас приложения

**Дата**: 2026-04-07
**Ветка**: `001-app-scaffold-auth`

## Сущности

### Profile

Основная сущность — профиль единственного пользователя приложения.

**Файл**: `{data_dir}/profile.json`

| Поле | Тип | Обязательное | Валидация | Описание |
|------|-----|:------------:|-----------|----------|
| `schemaVersion` | number | Да | `=== 1` | Версия схемы для будущей миграции |
| `pinHash` | string | Да | непустая, формат argon2id hash | Хэш пин-кода |
| `pinAlgorithm` | string | Да | `"argon2id"` | Алгоритм хэширования (для будущей миграции) |
| `createdAt` | string | Да | ISO 8601 | Дата создания профиля |
| `updatedAt` | string | Да | ISO 8601, ≥ `createdAt` | Дата последнего обновления |
| `profile` | object | Да | см. ниже | Данные пользователя |

**Вложенный объект `profile`**:

| Поле | Тип | Обязательное | Валидация | Описание |
|------|-----|:------------:|-----------|----------|
| `lastName` | string | Да | не пустая, 1–100 символов | Фамилия пользователя |
| `firstName` | string | Да | не пустая, 1–100 символов | Имя пользователя |
| `dateOfBirth` | string | Да | ISO 8601 (`YYYY-MM-DD`), не в будущем | Дата рождения |
| `sex` | string | Да | enum: `"male" \| "female"` | Пол |

**Пример**:

```json
{
  "schemaVersion": 1,
  "pinHash": "$argon2id$v=19$m=65536,t=3,p=4$...",
  "pinAlgorithm": "argon2id",
  "createdAt": "2026-04-07T10:30:00.000Z",
  "updatedAt": "2026-04-07T10:30:00.000Z",
  "profile": {
    "lastName": "Иванов",
    "firstName": "Иван",
    "dateOfBirth": "1990-05-15",
    "sex": "male"
  }
}
```

**Invariants**:
- Файл существует ⇔ пользователь зарегистрирован
- `pinHash` никогда не хранится в открытом виде
- При любом обновлении профиля обновляется `updatedAt`

### AppConfig

Конфигурационный файл приложения.

**Файл**: `{data_dir}/config.json`

| Поле | Тип | Обязательное | Значение по умолчанию | Описание |
|------|-----|:------------:|----------------------|----------|
| `schemaVersion` | number | Да | `1` | Версия схемы |
| `debug` | boolean | Да | `false` | Включить отладочное логирование |
| `dataDir` | string | Да | Путь к приложению | Директория для хранения данных (может быть переопределена) |

**Пример**:

```json
{
  "schemaVersion": 1,
  "debug": false,
  "dataDir": ""
}
```

## Схема расположения файлов

```
{APP_DATA_DIR}/
├── profile.json          # Профиль пользователя
├── config.json           # Конфигурация приложения
└── logs/
    └── app-YYYY-MM-DD_HH-mm-ss.log  # Лог-файл (создаётся при каждом запуске, при debug: true)
```

`APP_DATA_DIR` определяется через стандартные пути платформы:
- **Windows**: `%APPDATA%\my-health`
- **Linux**: `~/.local/share/my-health`
- **macOS**: `~/Library/Application Support/my-health`

Пути определяются через Tauri `path` API и конфигурируются через `config.json` → `dataDir`.

## Состояния и переходы

### Состояние приложения

```
[Не инициализировано]
        │
        ▼ (проверка existence profile.json)
        │
   ┌────┴────┐
   │         │
Нет профиль   Есть профиль
   │         │
   ▼         ▼
[Регистрация]  [Запрос пин-кода]
                 │
           ┌─────┴─────┐
           │           │
      Верный PIN   Неверный PIN
           │           │
           ▼           ▼
    [Основной экран]  [Ошибка, повтор]
```

### Валидационные правила

| Правило | Описание | Где применяется |
|---------|----------|----------------|
| PIN 4–6 цифр | Пин-код содержит от 4 до 6 цифровых символов | Регистрация, смена, вход |
| Дата не в будущем | `dateOfBirth ≤ today()` | Регистрация, редактирование профиля |
| Имя/фамилия не пустые | Минимум 1 символ, максимум 100 | Регистрация, редактирование профиля |
| Пол — enum | Только `"male"` или `"female"` | Регистрация, редактирование профиля |
| PIN совпадает | `pin === pinConfirm` | Регистрация, смена пин-кода |
| Текущий PIN верен | `argon2id.verify(storedHash, currentPin)` | Смена пин-кода |
