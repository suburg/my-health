# Быстрый старт: Модуль «Планы»

**Дата**: 2026-04-13
**Ветка**: `006-future-plans`

## Предварительные требования

- Установлен Node.js и Rust
- Проект `my-health` уже собран и запускается
- Модули «Приёмы», «Анализы», «Препараты» уже реализованы (паттерн-референс)

## Запуск

```bash
# Из корня проекта
npm run tauri dev
```

## Проверка модуля

1. Откройте приложение
2. В боковом меню выберите «Планы»
3. Убедитесь, что отображается пустое состояние «Добавьте первую задачу»
4. Нажмите «Добавить задачу» и создайте тестовую запись
5. Проверьте фильтрацию по виду, статусу и обязательности
6. Откройте детальную карточку, проверьте кнопки «Выполнить» и «Отменить»

## Тестовые данные

Для быстрого тестирования можно создать seed-данные вручную в файле:

```
%APPDATA%/com.myhealth.app/future-plans.json
```

Пример seed:

```json
{
  "schemaVersion": 1,
  "plans": [
    {
      "id": "test-1",
      "planType": "appointment",
      "plannedDate": "20.04.2026",
      "isMandatory": true,
      "description": "Плановый осмотр у терапевта",
      "status": "planned",
      "completedDate": null,
      "cancelReason": null,
      "createdAt": "2026-04-13T10:00:00.000Z",
      "updatedAt": "2026-04-13T10:00:00.000Z"
    },
    {
      "id": "test-2",
      "planType": "labTest",
      "plannedDate": "01.04.2026",
      "isMandatory": true,
      "description": "Общий анализ крови",
      "status": "planned",
      "completedDate": null,
      "cancelReason": null,
      "createdAt": "2026-04-13T10:00:00.000Z",
      "updatedAt": "2026-04-13T10:00:00.000Z"
    }
  ]
}
```

> Задача `test-2` с плановой датой в прошлом должна отображаться как «просроченная» (оранжевый индикатор).
