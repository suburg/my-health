## Сводка

Новый модуль «Анализы» для десктопного приложения my-health. Позволяет пользователю вводить и просматривать результаты лабораторных анализов (кровь, моча, биохимия, гормоны) с ручным вводом и LLM-распознаванием сканов. Ключевые возможности: реестр анализов, детальная карточка с историей показателей, нормализация названий к эталонному виду через встроенный справочник (~200 записей) + LLM, подсветка отклонений от референсов, поддержка числовых и текстовых показателей. Интерфейс аналогичен модулю «Приёмы» (Doctor Visits).

## Технический контекст

**Язык/Версия**: TypeScript 5.x (Frontend), Rust (Tauri backend)
**Основные зависимости**: React, shadcn/ui, Tailwind CSS, zod (валидация), @tauri-apps/api, существующий модуль doctor-visits как референс
**Хранилище**: JSON-файлы (`lab-tests.json`) + директория `scans/lab-tests/` для сканов
**Тестирование**: Vitest (unit-тесты frontend), cargo test (unit-тесты Rust)
**Целевая платформа**: Windows, Linux, macOS (Tauri desktop)
**Тип проекта**: Десктопное приложение (Tauri + React)
**Цели производительности**: Загрузка реестра <500мс при 500 записях, LLM-распознавание <30с на скан, рендеринг карточки <200мс при 50 показателях
**Ограничения**: Справочник ≤200 записей передаётся целиком в LLM-промпт; все данные локально; сканы до 10 МБ; поддержка числовых и текстовых показателей
**Масштаб/Объем**: 3 экранные формы (реестр, карточка, модалка), ~3 новых компонента, сервис, Tauri-команды, справочник показателей, unit-тесты

**Решение по структуре**: Один проект (Tauri + React). Модуль `lab-tests` следует паттерну `doctor-visits` — те же слои: компоненты → сервис → утилиты → типы → Rust-хранилище. Новые компоненты: `LabTestIndicatorTable` (динамическая таблица показателей), `LabTestBindDialog` (ручная привязка). Переиспользуется `ScanUploader` из doctor-visits.
