## Сводка

Новый модуль «Планы» для десктопного приложения my-health. Позволяет пользователю создавать и управлять плановыми задачами на будущее: визиты к врачу, исследования, анализы. Ключевые возможности: реестр задач с фильтрацией по виду, статусу и обязательности; детальная карточка с индикацией статуса; модальная форма создания/редактирования; действия «Выполнить» (с фактической датой) и «Отменить» (с опциональной причиной). Интерфейс аналогичен модулям «Приёмы», «Анализы» и «Препараты».

## Технический контекст

**Язык/Версия**: TypeScript 5.x (Frontend), Rust (Tauri backend)
**Основные зависимости**: React, shadcn/ui, Tailwind CSS, zod (валидация), @tauri-apps/api, существующие модули doctor-visits, lab-tests и medications как референсы
**Хранилище**: JSON-файлы (`future-plans.json`) в файловой системе
**Тестирование**: Vitest (unit-тесты frontend), cargo test (unit-тесты Rust)
**Целевая платформа**: Windows, Linux, macOS (Tauri desktop)
**Тип проекта**: Десктопное приложение (Tauri + React)
**Цели производительности**: Загрузка реестра <500мс при 500 записях, рендеринг карточки <200мс при 100 записях
**Ограничения**: Все данные локально; текстовые поля с лимитами (описание ≤ 500, причина отмены ≤ 300 символов)
**Масштаб/Объем**: 5 экранных форм (реестр, карточка, модалка создания/редактирования, модалка выполнения, модалка отмены), ~12 новых компонентов, сервис, Tauri-команды, утилиты, unit-тесты

│   └── __tests__/
│       └── future-plan-service.test.ts
├── lib/
│   ├── future-plan-utils.ts                # Фильтрация, сортировка, статусы
│   └── __tests__/
│       └── future-plan-utils.test.ts
├── lib/validations/

├── types/
│   └── index.ts                            # FuturePlan, FuturePlanType, FuturePlanStatus

│   └── future-plan-commands.ts             # IPC-команды (типы запросов/ответов)

├── src/

**Решение по структуре**: Один проект (Tauri + React). Модуль `future-plans` следует паттерну `medications` — те же слои: компоненты → сервис → утилиты → типы → Rust-хранилище. Нет внешних сканов/файлов — только JSON.
